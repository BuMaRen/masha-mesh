# 面试导向实践学习计划 TodoList

## 0. 计划原则（先做再讲）

- [ ] 每个主题都走完一轮：问题定义 -> 设计文档 -> 实现 -> 测试验证 -> 复盘文档
- [ ] 每个主题都产出可复用材料：1 份设计文档 + 1 份实现记录 + 1 份故障/边界复盘

### 1.2 每个主题的文档包（统一模板，最小可用）

- [ ] 设计文档：docs/<topic>-design.md（背景、目标、方案对比、流程、风险与回滚、验收标准）
- [ ] 实现记录：docs/<topic>-implementation-log.md（改动清单、提交顺序、设计偏差、已知问题）

- [ ] 先写设计初稿（先把方案对比和验收标准写清）

- [ ] 测试后补测试记录（至少覆盖 1 组异常路径）
- [ ] 最后写复盘（重点写误判、取舍和下一步）

## 3. 节奏建议（长期、低压力）

- [ ] 每 2 周完成 1 个主题闭环（设计 + 实现 + 复盘）
- [ ] 每周至少 1 次代码级回顾，更新文档中的“已知问题”
- [ ] 每个主题至少保留 1 个未完成优化项，作为后续面试延展点
- [ ] 每月做一次全局复盘：删除过时结论，补充新证据

## 4. 面试准备方式（不靠背诵）

- [ ] 每个主题准备 3 个你亲手解决的问题案例
- [ ] 每个案例固定回答结构：现象 -> 根因 -> 方案对比 -> 落地 -> 结果 -> 反思
- [ ] 每个案例至少有一个失败尝试，能讲清为何失败
- [ ] 每个案例有可量化指标或测试结论支持
